OnlyMapJS examples

Each page below is a real, runnable manifest — open it, pan/zoom/click around. They import the library straight from source (https://esm.sh/@nika-js/onlymap@0.2.0), the same way dev/index.html does, so they always reflect the current implementation.

Widgets

Legend, zoom-controls, scale-bar, and a live Vega-Lite chart that updates as you pan.

Behaviors & overlays

Click a region for a rich popup, hover a point for a tooltip, and drive a layer's filter with a slider.

Basemap

A real MapLibre basemap (no token) with data layers composited on top.

Columnar / GeoArrow

A week of real USGS earthquakes from an Arrow IPC file — same $field manifest, zero row objects on the render path, apache-arrow lazy-loaded on first use.

Draw

Sketch points, lines and polygons directly on the map with the draw toolbar — captured into a live layer and saved as a local GeoJSON file, no backend.

3D models

GLB models instanced on the basemap via ScenegraphLayer — the browser half of an IFC/CAD asset pipeline, no three.js required.

Google 3D Tiles

Google Photorealistic 3D Tiles rendered through Tile3DLayer. Requires a browser-restricted Map Tiles API key.

3D Flyby

A story-driven Google Photorealistic 3D Tiles flyby across nearby places, with dwell time to inspect tile loading and LOD refinement.

Live AIS

A live WebSocket ship feed (simulated, aisstream.io format) — upsert-by-key, throttled flushes, widgets updating per flush.

Fleet (polling)

Drivers from an auth-protected REST snapshot endpoint, re-polled with refresh="1s" — the Grab-style transport, with configureData supplying the bearer token.

Map story

A guided tour as a manifest: om-story steps fire the same actions behaviors use, with a player widget, seek/scrub, and pause-on-grab.

React adapter

The programmatic front-end as React components: OmMap, OmLayer, OmWidget, OmOverlay, and useOmMap().
